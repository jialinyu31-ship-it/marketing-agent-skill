# Industry Module Framework

Use this framework with any file under `references/industries/`. Load the specific industry file first for audience, emotional tension, trust barriers, topic clusters, and routing boundaries; then apply this shared workflow.

## Routing Discipline

Before using an industry module:

1. Confirm whether the user's request is about an industry, an audience group, a life stage, a product category, or a regulated service.
2. Check the module's `Use When` and `Do Not Use When` sections.
3. If two modules could apply, choose the narrower commercial intent:
   - product/service category beats broad audience group;
   - regulated service beats general education/opinion content;
   - lifecycle stage beats generic industry label.
4. If ambiguity remains, state the ambiguity and proceed with the closest module plus one backup module.

## Content Workflow

Build vertical content in this order:

1. `Scene`: name a concrete buying, comparing, usage, or anxiety scene.
2. `Pressure`: identify the emotional tension without blaming the audience.
3. `Decision Rule`: reveal one checklist, comparison criterion, or expert standard.
4. `Proof`: add process, case boundary, receipt, demo, common failure mode, or qualification note.
5. `Action`: guide to comment, save, diagnose, request a quote, book a call, visit a store, or compare options.

## Short-Video Script Pattern

Use this structure for most 30-90 second口播 videos:

| Segment | Requirement |
| --- | --- |
| Opening | one concrete mistake, scene, contradiction, or decision pressure |
| Empathy | name the audience's fear, desire, shame, urgency, or uncertainty |
| Turn | explain why the common judgment standard is incomplete |
| Rule | give one practical decision rule or self-check |
| Proof | show the basis without overclaiming |
| CTA | request the lowest-friction next action |

## Private-Domain Handoff

- First reply: acknowledge the exact topic the user commented on and offer the relevant checklist or diagnostic question.
- Segment by stage: researching, comparing, urgent, after a bad experience, ready to buy.
- Ask no more than 3 qualification questions before giving value.
- Move qualified leads to consultation, quote, store visit, audit, plan draft, or professional referral.
- Record repeated objections as future content topics.

## Output Fields

For vertical topic planning, output:

| Field | Meaning |
| --- | --- |
| 选题 | publishable topic |
| 适用模块 | selected industry file and any backup module |
| 目标人群 | exact audience and decision stage |
| 情绪触发 | anxiety, curiosity, relief, regret, identity, control, or trust |
| 冲突点 | what belief, role, cost, or risk collides |
| 前三秒钩子 | first sentence or title |
| 正文推进 | 3-5 step narrative or argument flow |
| 信任证据 | process, case, standard, document, demo, boundary |
| 评论/私信引导 | low-friction next action |
| 合规风险 | claim, proof, qualification, privacy, platform risk |

## Prompt Skeleton

```text
你是[行业/人群]的中文营销策划助手。
请为[产品/服务]面向[目标人群]生成[数量]条短视频选题。
先判断应该加载哪个行业模块，并说明选择理由。
每条必须包含：选题、目标人群、情绪触发、冲突点、前三秒钩子、正文推进、信任证据、评论/私信引导、合规风险。
请优先使用具体场景、真实决策压力和可执行自查标准；不要复制来源提示词原文，不要编造数据或案例。
```

## Compliance Gate

For finance, investment, lending, debt, healthcare, medical beauty, law, education, insurance, psychological counseling, employment placement, adult products, and eldercare:

- avoid guaranteed outcomes;
- avoid fear exploitation;
- separate education from professional advice;
- require source verification for numbers, laws, policy, pricing, medical, legal, or investment claims;
- route high-risk cases to qualified professionals or emergency support when appropriate.
