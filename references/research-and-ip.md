# Research And IP Workflow

Use this file when the user needs industry analysis, audience insight, personal IP positioning, competitor review, or a reusable marketing agent prompt.

## Intake

Collect or infer the following:

| Field | What to Capture | Why It Matters |
| --- | --- | --- |
| 行业/细分赛道 | category, niche, geography, channel | Sets comparison frame and compliance limits |
| 产品/服务 | offer, price, delivery method, proof | Determines trust barrier and conversion path |
| 目标人群 | identity, stage, pain, desire, budget | Controls language, cases, and CTA |
| 竞争环境 | direct competitors, substitutes, content accounts | Shows differentiation and overused angles |
| 成交链路 | platform, comment, DM, WeChat, call, store | Converts content into action |
| 品牌人格 | expert, founder, operator, buyer friend, challenger | Controls voice and story material |

When inputs are incomplete, create a visible assumption block and keep the output editable.

## Industry Insight Stack

Analyze an industry through four layers:

1. `事实层`: market category, buyer roles, common purchase triggers, price bands, service steps, after-sales concerns.
2. `认知层`: what customers misunderstand, what insiders know but customers do not, what competitors avoid explaining.
3. `情绪层`: shame, anxiety, status desire, fear of waste, fear of being cheated, need for control, need for identity.
4. `行动层`: what the audience can do after watching one piece of content: comment, self-check, save, compare, consult, buy.

Convert every insight into a content opportunity:

| Insight Type | Content Conversion |
| --- | --- |
| Misunderstanding | "Most people judge X by A, but the real variable is B." |
| Hidden process | Behind-the-scenes explanation, checklist, timeline, cost breakdown |
| Trust barrier | Proof, comparison, objection handling, failure case |
| Emotional pressure | POV topic, confession-style opening, role conflict, myth-busting |
| Low action clarity | Diagnostic questions, calculator, template, decision tree |

## Personal IP Biography Interview

Use this when building founder IP, expert IP, consultant IP, trainer IP, or local-service owner IP.

Interview by identity layer. Ask one question at a time, then use follow-ups to extract concrete scenes and language.

| Identity Layer | Interview Goal | Useful Prompts |
| --- | --- | --- |
| 工作身份 | expertise, cases, method, hard-won lessons | "What problem do clients repeatedly misunderstand?" |
| 家庭身份 | values, responsibility, emotional grounding | "Which family moment changed how you make business decisions?" |
| 社交身份 | reputation, peer perception, community role | "What do friends ask you for help with most often?" |
| 自由身份 | personal quirks, beliefs, contradictions | "What unpopular belief do you keep because experience proved it right?" |

Transform answers into IP material:

- `人物底色`: early experience, recurring choice, emotional motivation.
- `专业标签`: method, category, proof, repeatable diagnostic lens.
- `反差标签`: unexpected contrast that makes the person memorable.
- `故事资产`: moments with conflict, cost, decision, consequence.
- `商业承接`: which stories naturally lead to consultation, course, service, or product.

Do not over-polish the biography. Preserve specific details, spoken rhythm, and contradictions.

## IP Concept Generator

Create a compact IP concept before creating content:

1. `一句话定位`: "I help [audience] solve [pain] through [method] without [common fear]."
2. `三类可信证据`: personal history, client/process proof, public artifact.
3. `四个内容支柱`: education, case, opinion, behind-the-scenes.
4. `一个稳定冲突`: expert truth versus market misconception.
5. `一个商业入口`: diagnostic, consultation, lead magnet, product trial, store visit.

Output format:

| Module | Draft |
| --- | --- |
| IP name/label | 3-5 candidate labels |
| Audience | primary and secondary |
| Promise | clear but not guaranteed |
| Differentiation | what this IP refuses to do or say |
| Content pillars | weekly repeatable topics |
| Monetization path | soft CTA and direct CTA |

## 4x4 Competitor Analysis

Use four competitors and four dimensions. If fewer competitors are known, include substitute choices such as search results, leading accounts, local players, or offline alternatives.

| Dimension | Questions |
| --- | --- |
| Positioning | Who do they serve, what promise do they repeat, what status do they signal? |
| Content | Which formats, hooks, proof types, and posting rhythm do they rely on? |
| Offer | What do they sell, how do they price, what objections do they answer? |
| Weakness | Where are they vague, overclaiming, repetitive, or emotionally disconnected? |

Turn gaps into actions:

- If competitors are too expert-heavy, create customer-decision content.
- If competitors are too emotional, create proof and process content.
- If competitors focus on price, create risk-cost and wrong-choice content.
- If competitors use generic advice, create vertical-specific scenarios.

## Commercial IP Presentation Modes

Choose formats by audience maturity:

| Mode | Best For | Structure |
| --- | --- | --- |
| Diagnostic explainer | cold audience with unclear problem | symptom -> cause -> self-check -> next step |
| Case teardown | warm audience seeking proof | scene -> decision -> process -> result -> lesson |
| Debate/opinion | saturated category | common claim -> disagreement -> evidence -> boundary |
| Founder story | trust building | conflict -> choice -> cost -> principle -> current offer |
| Behind-the-scenes | high-trust service | process -> standard -> failure prevention -> CTA |
| Dialogue | objection handling | customer worry -> expert clarification -> decision rule |
| Data/receipt | skeptical buyers | claim -> metric/document -> interpretation -> caveat |
| Immersive POV | high-emotion category | identity -> pressure -> inner conflict -> relief path |

## Research Prompt Skeleton

Use this skeleton to create new agent prompts:

```text
你是[角色]，任务是为[行业/产品]分析[目标人群]的营销机会。
输入包括：[已知信息]。
请按以下步骤工作：
1. 明确用户问题、购买场景和决策风险。
2. 拆解行业事实、用户误区、情绪压力和信任障碍。
3. 找出内容机会、差异化表达和商业承接点。
4. 输出可执行表格，并标注假设、风险和需要用户补充的信息。
输出格式：[表格字段]。
限制：不编造数据，不作保证性承诺，不复制来源材料原文。
```
