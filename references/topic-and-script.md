# Topic And Script Workflow

Use this file for short-video topic planning, emotional POV generation, story scripts, hooks, hot-topic entry, shot planning, and script prompt creation.

## Emotional POV Topic Engine

Every strong topic should connect a business point with a human pressure.

1. `问题边界`: who has the problem, when it appears, what decision is at stake.
2. `常规解法`: what most people do and why it feels reasonable.
3. `代价`: money, time, identity, status, safety, trust, opportunity.
4. `反向观点`: what the creator believes that the audience does not yet believe.
5. `情绪触发`: anxiety, anger, relief, pride, regret, curiosity, unfairness.
6. `行动出口`: comment keyword, save checklist, compare options, ask for plan, buy/consult.

Topic table:

| Field | Question |
| --- | --- |
| 选题 | What is the viewer likely to click or save? |
| 目标人群 | Which identity and situation is this for? |
| 情绪钩子 | What feeling appears in the first 3 seconds? |
| 冲突 | Which belief, role, or interest is colliding? |
| 反常识 | What will surprise a qualified buyer? |
| 证据 | Which case, process, receipt, or comparison supports it? |
| CTA | What action should happen next? |

## POV Formulas

Use these as structures, not as fixed copy.

| Formula | When To Use | Structure |
| --- | --- | --- |
| 公敌式 | category has a disliked practice | name the practice -> explain harm -> offer a fairer rule |
| 反向思维 | audience follows a weak habit | common advice -> hidden cost -> better criterion |
| 共情式 | audience feels blamed or confused | name the feeling -> validate context -> give next step |
| 金句式 | creator needs memorable authority | sharp belief -> example -> rule -> application |
| 奇葩问题式 | topic needs novelty | unusual question -> unexpected logic -> useful answer |
| 热点切入式 | a trend connects to the niche | public event -> buyer lesson -> category-specific advice |
| 内心戏式 | high anxiety decisions | inner conflict -> feared consequence -> decision rule |
| 造谣反转式 | playful rumor/debunk content | exaggerated assumption -> reveal truth -> practical point |

## Hook Design

Create 3-5 hooks per topic. Test each hook against these criteria:

- It names a concrete person, scene, mistake, or decision.
- It creates tension without misleading the viewer.
- It can be understood without industry jargon.
- It leads naturally into the promised content.
- It avoids exaggerated guarantees.

Hook patterns:

| Pattern | Example Shape |
| --- | --- |
| Mistake | "If you choose X only by Y, you may miss Z." |
| Time pressure | "Before you sign/pay/choose, check these three things." |
| Identity | "This is for people who are good at A but always get stuck at B." |
| Insider rule | "In this industry, professionals do not first look at A; they look at B." |
| Cost reversal | "The expensive mistake is often not buying high, but buying the wrong standard." |

## Script Structure

Use this default structure for 30-90 second videos:

1. `黄金三秒`: name the pain, conflict, or counterintuitive rule.
2. `场景复现`: show the viewer where the issue appears.
3. `错误路径`: explain what people usually do and why it fails.
4. `关键洞察`: reveal the decision variable or mental model.
5. `证据`: case, comparison, process, quote, receipt, demonstration, or before/after.
6. `行动`: comment, save, ask, compare, or book a diagnosis.

For story-based scripts:

| Segment | Content |
| --- | --- |
| Identity | who is speaking and why viewers should care |
| Pressure | conflict, shame, urgency, unfairness, or tradeoff |
| Decision | what changed and what it cost |
| Detail | concrete object, number, scene, sentence, or behavior |
| Insight | rule that can transfer to the audience |
| Commercial bridge | why this connects to the service/product |

| Goal | Formats |

## Hot-Topic Entry

Use hot topics only when there is a credible bridge to the niche.

1. Identify the public topic's core conflict.
2. Translate it into a buyer lesson or business lesson.
3. State the boundary: what does and does not apply to this niche.
4. Create one content piece for attention and one for conversion.

Avoid trend hijacking when the event involves tragedy, private individuals, sensitive identity, or unverified claims.

## Script Prompt Skeleton

```text
你是短视频内容策划和脚本导演。
请基于[行业/产品/人群/目标]生成[数量]条短视频脚本。
每条脚本必须包含：
- 选题标题
- 目标人群和观看场景
- 前3秒钩子
- 口播正文
- 镜头或画面建议
- 信任证据
- 评论/私信/成交引导
- 风险和合规备注
要求：口语化、具体、有冲突；不要夸大承诺；不要复制任何来源提示词原文。
```
